<DocSection title="The Simple Model">

Ghost captures a project's product experience memory in the repo, where your
agent can use it. The public package is `@anarchitecture/ghost`, and it
installs one CLI: `ghost`.

The canonical fingerprint is small:

```text
.ghost/fingerprint.yml -> .ghost/checks.yml
memory and why            deterministic gates
```

`intent.md`, `decisions/*.yml`, and `proposals/*.yml` are optional
human-approved or staged context. `cache/` can hold generated inventory, but it
is not canonical memory.

| Surface | Job | Verbs |
| --- | --- | --- |
| `ghost` | Create and check the root `.ghost/` memory bundle, review diffs, compare bundles, and record drift intent. | `init`, `scan`, `inventory`, `lint`, `verify`, `describe`, `diff`, `survey <op>`, `check`, `review`, `compare`, `ack`, `track`, `diverge`, `emit`, `skill install` |
| `ghost-fleet` | See how many project fingerprints relate. | `members`, `view`, `emit skill` |
| `ghost-ui` | Reference design system Ghost uses to test itself. | - |

</DocSection>

<DocSection title="Install Ghost">

```bash
npm install -D @anarchitecture/ghost
npx ghost --help
npx ghost skill install
```

Once the skill is installed, ask your agent in plain English: "Capture a Ghost
fingerprint for this repo" or "review this PR for drift". The recipe tells the
agent what to read, what to write, and which CLI checks to run.

</DocSection>

<DocSection title="Capture Your First Fingerprint">

```text
Capture a Ghost fingerprint for this repo.
```

Fingerprint Capture writes durable memory first:

1. **Summary and topology** - what product this is, who it serves, and where UI
   surfaces live.
2. **Situations** - user, task, and state moments that change obligations.
3. **Principles and contracts** - durable product judgment plus behavior,
   disclosure, failure, and recovery rules.
4. **Patterns and implementation vocabulary** - accepted visual, content,
   behavior, and composition memory plus current tokens and components agents
   may use to implement it.

```bash
ghost init --with-intent
ghost scan --format json
ghost inventory > .ghost/cache/inventory.json
ghost lint .ghost
ghost verify .ghost --root .
```

Inventory is optional source material. Promote only durable conclusions into
`fingerprint.yml`.

</DocSection>

<DocSection title="Compare Two Systems">

```bash
ghost compare market/.ghost dashboard/.ghost
ghost compare */.ghost
```

Bundle comparison uses canonical Ghost memory when available. Direct
fingerprint markdown files still work for legacy semantic comparison:

```bash
ghost compare a.fingerprint.md b.fingerprint.md --semantic
```

</DocSection>

<DocSection title="Review Drift in a PR">

```text
Review this PR for design drift
```

`ghost check` applies active deterministic gates from `.ghost/checks.yml`.
`ghost review` emits advisory context grounded in `.ghost/fingerprint.yml`,
optional `.ghost/intent.md`, open proposals, checks, and the diff. Add
`--include-memory` to include accepted `.ghost/decisions/*.yml` in the
advisory packet.

</DocSection>

<DocSection title="Propose Fingerprint Updates">

Use the installed `ghost` skill when work reveals memory Ghost should consider
adding to the fingerprint:

```text
Brief this work from the Ghost fingerprint
Propose this product-experience decision as a fingerprint update
Promote this accepted proposal
```

The recipes write proposals first. Humans promote durable context into
`fingerprint.yml` or `checks.yml`.

</DocSection>

<DocSection title="Track Intent Toward a Direct Fingerprint">

Drift without intent is noise; drift with intent is signal. `ack`, `track`, and
`diverge` still operate on tracked direct fingerprint markdown files:

```bash
ghost ack --stance aligned --reason "Initial baseline"
ghost track new-tracked.fingerprint.md
ghost diverge typography --reason "Editorial product uses a different type scale"
```

</DocSection>
