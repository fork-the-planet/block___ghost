<DocSection title="Overview">

The CLI does the repeatable parts: initialize memory bundles, report capture
progress, validate files, emit raw inventory, compare bundles, check diffs,
emit handoff packets, and record intent. Your agent does the reading, writing,
and reviewing.

Canonical Ghost memory starts here:

```text
.ghost/fingerprint.yml
.ghost/checks.yml
```

Install the public package and one skill bundle:

```bash
npm install -D @anarchitecture/ghost
npx ghost skill install
```

Ask your agent to run the workflow:

```text
Capture a Ghost fingerprint for this repo.
```

Commands are grouped by job:

- **Create and check the memory bundle**: `init`, `scan`, `inventory`, `lint`, `verify`, `describe`, `diff`, `survey <op>`, `emit`.
- **Review and compare drift**: `check`, `review`, `compare`, `ack`, `track`, `diverge`.
- **Install BYOA recipes**: `skill install`.
- **View many projects**: `ghost-fleet members`, `ghost-fleet view`, `ghost-fleet emit skill`.

Workflows like _capture_, _survey_, _patterns_, _recall_, _brief_, _critique_,
_propose_, _promote_, _review_, _verify_, _compare_, and _remediate_ are skill
recipes your host agent runs. They are installed by `ghost skill install`; they
are not LLM-running CLI verbs.

The tables below are generated from each CLI's source at build time. If a flag
changes in any `cli.ts`, the next `pnpm dump:cli-help` run regenerates this
reference.

</DocSection>

<DocSection title="ghost - capture support and bundle checks">

`ghost` supports Fingerprint Capture. It inventories the repo, reports capture
progress, validates memory, verifies evidence paths and typed refs, describes
legacy direct markdown when needed, diffs direct fingerprint files, runs
legacy/cache survey ops, and emits agent-ready outputs.

### Initialize - `init`

Create a `.ghost/` memory skeleton with `fingerprint.yml`, `checks.yml`,
`proposals/`, and `cache/`.

<CliHelp tool="ghost" command="init" hideDescription />

```bash
ghost init
ghost init --with-intent
```

### Capture progress - `scan`

Report whether fingerprint memory is present, whether it has accepted entries,
which optional files exist, and which BYOA step the agent should run next.
`--format json` is the deterministic handoff.

<CliHelp tool="ghost" command="scan" hideDescription />

```bash
ghost scan
ghost scan ../other-repo/.ghost
ghost scan . --format json
ghost scan . --include-scopes
```

### Inventory cache - `inventory`

Emit raw signals about a frontend repo as JSON: package manifests, language
histogram, candidate config files, registry presence, top-level tree, and git
remote. Use this as optional source material, usually under `.ghost/cache/`.

<CliHelp tool="ghost" command="inventory" hideDescription />

```bash
ghost inventory
ghost inventory ../other-repo > .ghost/cache/inventory.json
```

### Validation - `lint`

Validate a root `.ghost` bundle or an individual artifact against its schema.
Use this before declaring a file structurally valid; every authoring recipe
calls it. Follow `lint` with `verify` to check evidence and references.

<CliHelp tool="ghost" command="lint" hideDescription />

```bash
ghost lint
ghost lint .ghost/fingerprint.yml
ghost lint .ghost/checks.yml --format json
```

### Bundle fidelity - `verify`

Validate that a root `.ghost` bundle is internally faithful:
`fingerprint.yml` evidence paths should resolve from `--root`, typed check refs
must point at known checks, and active checks must be grounded in fingerprint
memory.

<CliHelp tool="ghost" command="verify" hideDescription />

```bash
ghost verify .ghost --root .
ghost verify .ghost --root . --format json
```

### Legacy survey ops - `survey <op>`

Operate on `ghost.survey/v2` files as legacy/cache source material.

- `merge` combines surveys with id-based dedup.
- `fix-ids` recomputes every row's `id` from its content.
- `summarize` prints a bounded Markdown or JSON digest for migration.
- `catalog` prints exact value enums/specs derived from survey rows.
- `patterns` derives a legacy `ghost.patterns/v1` draft from surface evidence.

<CliHelp tool="ghost" command="survey" hideDescription />

```bash
ghost survey merge a.json b.json -o merged.json
ghost survey fix-ids draft.json -o draft.json
ghost survey summarize survey.json
ghost survey catalog survey.json --kind color
```

### Inspection - `describe`

Print optional `.ghost/intent.md` or a section map of a direct fingerprint
markdown file. The host agent uses this to load only the sections it needs.

<CliHelp tool="ghost" command="describe" hideDescription />

```bash
ghost describe
ghost describe path/to/fingerprint.md
ghost describe --format json
```

### Structural diff - `diff`

Diff two legacy direct fingerprints at the prose and structural level:
decisions, palette roles, and tokens. This is not vector distance; use
`ghost compare` for that.

<CliHelp tool="ghost" command="diff" hideDescription />

```bash
ghost diff a/fingerprint.md b/fingerprint.md
ghost diff a.md b.md --format json
```

### Derived outputs - `emit`

Emit deterministic artifacts for agents and editors. Supported kinds are
`review-command` and `context-bundle`; both default to `.ghost/fingerprint.yml`
memory. Explicit `--fingerprint` keeps legacy direct markdown mode available.
Skill installation lives at `ghost skill install`.

<CliHelp tool="ghost" command="emit" hideDescription />

```bash
ghost emit review-command
ghost emit context-bundle
ghost emit context-bundle --prompt-only
ghost emit context-bundle --out dist/context
```

</DocSection>

<DocSection title="ghost - review and compare">

### Deterministic gates - `check`

Run active `ghost.checks/v1` gates from `.ghost/checks.yml` against a git diff.

<CliHelp tool="ghost" command="check" hideDescription />

```bash
ghost check --base main
ghost check --diff patch.diff --format json
```

### Advisory packet - `review`

Emit an evidence-routed advisory review packet grounded in
`.ghost/fingerprint.yml`, optional `.ghost/intent.md`, open proposals, accepted
decisions, checks, and the diff.

<CliHelp tool="ghost" command="review" hideDescription />

```bash
ghost review --base main
ghost review --base main --include-memory --format json
```

### Comparison - `compare`

Pairwise distance (N=2) or composite analysis (N>=3) over root bundles or direct
fingerprint markdown. Bundle inputs use canonical Ghost memory when available.
Direct markdown inputs still support `--semantic` and `--temporal`.

<CliHelp tool="ghost" command="compare" hideDescription />

```bash
ghost compare market/.ghost dashboard/.ghost
ghost compare a.fingerprint.md b.fingerprint.md --semantic
ghost compare before.fingerprint.md after.fingerprint.md --temporal
ghost compare */.ghost
```

### Intent - `ack` / `track` / `diverge`

These three verbs write per-dimension stances to `.ghost-sync.json`. `ack` and
`diverge` need a tracked fingerprint declared in `ghost.config.ts`; `track`
takes the new tracked fingerprint as its argument.

<CliHelp tool="ghost" command="ack" hideDescription />

```bash
ghost ack --stance aligned --reason "Initial baseline"
ghost ack -d typography --stance diverging --reason "Brand refresh requires different type scale"
```

<CliHelp tool="ghost" command="track" hideDescription />

```bash
ghost track new-tracked.fingerprint.md
```

<CliHelp tool="ghost" command="diverge" hideDescription />

```bash
ghost diverge palette --reason "Dark-mode-first palette for this product"
```

### Skill bundle - `skill install`

Install the unified `ghost` skill bundle into a host agent. It contains
Fingerprint Capture, survey/cache, patterns, schema, recall, brief, critique,
propose, promote, review, verify, compare, and remediate recipes.

<CliHelp tool="ghost" command="skill" hideDescription />

```bash
ghost skill install
ghost skill install --agent codex --force
ghost skill install --dest ~/.my-agent/skills/ghost
```

</DocSection>

<DocSection title="ghost-fleet - view across projects">

A fleet is a directory containing many projects, each with a Ghost fingerprint
bundle. `ghost-fleet` reads the fleet, emits per-fleet outputs (`fleet.md` and
`fleet.json`), and ships the agentskills.io bundle for the narrative recipe.

<CliHelp tool="ghost-fleet" command="members" hideDescription />

```bash
ghost-fleet members
ghost-fleet members --json
```

<CliHelp tool="ghost-fleet" command="view" hideDescription />

```bash
ghost-fleet view ./fleet
ghost-fleet view ./fleet --id cash-mobile --out reports/cash
```

<CliHelp tool="ghost-fleet" command="emit" hideDescription />

```bash
ghost-fleet emit skill
```

</DocSection>

<DocSection title="Skill recipes">

These are instruction files inside the unified skill bundle. Install them once,
then ask your agent in plain English.

| Recipe | Bundle | Trigger |
| --- | --- | --- |
| `capture` | `ghost` | "Capture a Ghost fingerprint for this repo" |
| `survey` | `ghost` | "use survey cache" / "extract tokens as source material" |
| `patterns` | `ghost` | "write fingerprint patterns" / "codify product-experience patterns" |
| `recall` / `brief` / `critique` / `propose` / `promote` | `ghost` | "brief this work" / "propose this decision" / "promote this proposal" |
| `review` | `ghost` | "review this PR for drift" |
| `verify` | `ghost` | "verify generated UI against the bundle" |
| `compare` | `ghost` | "why did these two fingerprints drift?" |
| `remediate` | `ghost` | "fix this drift" |
| `target` | `ghost-fleet` | "describe this fleet" |

Source lives under `packages/ghost/src/skill-bundle/references/`. The agent
executes the steps with its normal tools and calls the relevant CLI command
when it needs a reproducible answer.

</DocSection>
